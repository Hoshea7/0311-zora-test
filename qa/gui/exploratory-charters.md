# Exploratory Charters

探索式 GUI 测试不是随便乱点，而是 Codex 带着目标模拟真实用户，寻找剧本没有覆盖到的体验断点。

## Charter 1: First-Run Friction

目标：找出首次用户从打开应用到完成首轮对话的所有摩擦。

探索动作：

- 不看代码，只按界面提示行动。
- 尝试空字段、错误 API key、切换 Provider 类型。
- 观察错误文案是否可理解。
- 观察是否有死路或无法返回。

输出：

- 首次启动流程中的阻塞点。
- 文案/交互不清楚的地方。
- 可沉淀为产品规则与验收准则的发现。

## Charter 2: Agent Conversation Feel

目标：验证 Zora 作为 Agent 的对话体验是否自然、可观察、可恢复。

探索动作：

- 发送短问题、长问题、中文问题。
- 等待真实 SDK 回复。
- 观察 thinking、流式消息、错误恢复。
- 切换 session 后回来观察上下文是否保留。

输出：

- Agent 等待期间是否有足够反馈。
- 回复失败是否能恢复。
- session 状态是否和用户预期一致。

## Charter 3: Settings Robustness

目标：确认设置页不会让用户迷路。

探索动作：

- 进入模型配置、记忆设置、技能设置。
- 在设置页和聊天页之间切换。
- 尝试关闭、保存、取消、重复打开。

输出：

- 状态是否同步。
- 保存后是否有明确反馈。
- 设置失败是否可恢复。
